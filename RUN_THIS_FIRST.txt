STEP 2: TEST THE FIX
====================

After the build completes, test it:

    cd dist\WindowSupport\WindowPowerShellProvider

STEP 3: VERIFY STARTUP REGISTRATION
====================================

Check if the scheduled task was created:

    schtasks /query /tn "WindowSupportMonitor"

You should see the task with Status: Ready

WHAT WAS CHANGED:
=================

1. Changed from --onefile to --onedir mode in all .spec files
   - This prevents the PyInstaller extraction errors
   - More stable and reliable

2. Improved startup registration in WindowPowerShellProvider.py
   - Better error reporting
   - Fallback to startup folder if task creation fails
   - Detailed logging of all steps
